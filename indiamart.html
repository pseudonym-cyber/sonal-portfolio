Phase 1 · Buyer Intelligence Platform

Buyer Memory

IndiaMART has solved supplier discovery. The next opportunity is buyer continuity — every procurement becomes context for the next one.

Problem

The buyer journey ends at the enquiry

IndiaMART has excellent visibility until a buyer sends an RFQ. After that, procurement moves to calls, WhatsApp and email, and every future purchase starts again from zero — even though years of search, RFQ and supplier-interaction history already sit inside the platform, mostly unused.

Insight

The opportunity isn't transactions. It's memory.

IndiaMART doesn't need to own the transaction to own the intelligence. Search history, RFQ history and supplier-interaction history are all data IndiaMART already has — enough to make every next procurement faster without waiting on data it doesn't and may never control.

Decision

Buyer Memory — a platform, not a feature

Buyer Memory is a single layer that every screen reads from and writes to — search, RFQ, supplier listing, notifications. Nothing else is redesigned. It shows up as a small set of Phase 1 capabilities:

  • Procurement Workspace — active requirements as ongoing entities, not disposable RFQs
  • Smart Reordering — surfaces recurring purchases before the buyer has to remember them
  • Requirement Reuse — checks history before showing a blank form
  • Recommendations — always explained, never a bare "recommended" label
  • Market Intelligence — trend signals tied to the buyer's own active procurements
  • Suppliers You Know — quick access by engagement frequency, not a vendor-management system
Trade-off

What this deliberately does not try to be

Several adjacent ideas were considered and set aside for Phase 1 — not because they're wrong, but because they need something Buyer Memory has to build first:

IdeaWhy not yet
Vendor ManagementNeeds buyer-supplier relationship data this platform hasn't accumulated yet
Procurement TrackingIndiaMART doesn't own transaction completion, delivery or order value
AI chat / negotiationWithout memory, AI has no context — it becomes a generic chatbot
ERP-style featuresOutside IndiaMART's positioning; the platform should stay lightweight
The safeguard that matters most: personalization is capped so it never quietly starves new-supplier discovery. "Previously Engaged" suppliers are capped at 40% of listing slots, and "Explore Marketplace" always keeps at least 2 slots above the fold — because supplier subscriptions still depend on new-buyer reach, and that's the one trade-off Buyer Memory cannot get wrong.
Measurement

What changes, and for whom

The platform only matters if it moves something on all three sides of the marketplace — not just buyer convenience:

Buyer
  • Fewer fake / low-intent enquiries to sort through
  • Less cognitive load — memory does the repeat work
  • Higher trust in IndiaMART as a sourcing partner
Supplier
  • Fewer junk enquiries, more relevant ones
  • Higher buyer retention on the platform
  • Better buyer-supplier match quality
IndiaMART
  • Buyer retention and supplier renewal, both up
  • Organised behavioural data → faster decisions
  • Compounding asset: better data makes the AI better

If IndiaMART could track only one number to know whether this is actually working, it wouldn't be enquiry count — the very metric the platform runs on today is the one this proposal argues against optimizing the buyer experience for:

Optimizes today for
Number of enquiries
→
Should optimize for
Time to confident supplier selection

This is a proxy, not a confirmed outcome — IndiaMART can't see whether a purchase actually completed offline. It's tracked alongside a Lead Concentration Index, which watches the 40% cap above and is the metric that would catch this platform quietly hurting new-supplier reach before it became a problem. Note this is about what drives buyer-facing product decisions, not about abandoning enquiry volume as a business metric — that's exactly what the Lead Concentration Index protects.

Prototype

How to read the working prototype

  1. Persona toggle (top left) — switch between a returning buyer with history and a brand-new buyer to see the cold-start fallback.
  2. Buyer Experience tab — everything is real and clickable: continue a procurement, shortlist and mark suppliers won, post a requirement, reorder, dismiss a recommendation.
  3. Platform View tab — "This session" metrics genuinely update from what you just clicked; "At marketplace scale" metrics are illustrative example numbers, not computed from the demo, and are labeled as such.

The prototype is built to demonstrate product and platform thinking, not final visual design — treat the interaction model as the artifact, not the pixels.
